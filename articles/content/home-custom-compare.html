ショートカット・Webアプリ方式・OJapp方式を徹底比較

iPhone / Android の「ホーム画面に追加」には、実は大きく分けて3つの方法があります。

  • ① ショートカット方式（iPhone標準のカスタム）
  • ② Webアプリの標準「ホーム画面に追加」機能
  • ③ OJapp方式（自由アイコンで配置できる）

この記事では、「どの方法で何ができて、何ができないか」を分かりやすく整理します。

① ショートカット方式とは？

iPhone の「ショートカット」アプリを使い、   任意の画像＋任意の名称で “疑似アプリ” を作成する方法です。

メリット

  • アイコンを完全に自由にできる
  • 名称も好きな文字にできる
  • 「アプリをテーマに合わせて統一したい」人向け

デメリット

  • 設定が複雑で初心者には難しい
  • 作ったものを共有できない（相手に手順が必要）
  • 起動時にショートカット画面が一瞬表示される
  • 環境ごとに作り直しになる（スマホ変更時など）

→ 自分のために使う分には良いが、人に勧めるにはハードルが高い。

② Safari / Chrome の標準「ホーム画面に追加」

最もシンプルなやり方で、iPhone / Android どちらでも使えます。

メリット

  • 数秒で追加できる
  • 起動が高速（ショートカットのような遅延がない）
  • 誰でも説明なしで使える

デメリット

  • アイコンは “サイト側が設定した画像” に固定される
  • note → n、BOOTH → ラクダ、BASE → 三角ロゴ などになる
  • Webサイト側に依存するので自由度は低い

→ 手軽だが、「自分のアイコンで置いてもらう」用途には向かない。

③ OJapp方式とは？

アイコン・アプリ名・URL をすべて自由に設定した Web ページを生成し、   それをホーム画面に追加する方式です。

メリット

  • 好きなアイコンを使える
  • シェアURLだけで誰でも再現できる（共有性が最強）
  • ショートカット不要
  • iPhone / Android 両対応
  • 起動も速い（ショートカットの遅延なし）

デメリット

  • Webツールを一度開く必要はある
  • アイコン画像を用意する必要がある

→ 「人に配る」「ショップや作品サイトを置いてもらう」用途では最強。

3方式の比較表

項目 ショートカット 標準ホーム追加 OJapp方式
設定の簡単さ 難しい とても簡単 簡単
アイコン自由度 完全自由 サイト依存 自由
共有のしやすさ 共有不可 URL共有は不可 URLで即共有できる
起動速度 やや遅い 速い 速い
スマホ変更時の移行 作り直し 簡単 URLを開くだけ

目的別のおすすめ

■ 見た目重視でアイコンを統一したい → ショートカット方式
■ とにかく時短で置きたい → Safari / Chrome の標準方式
■ ショップやポートフォリオを人に置いてほしい → OJapp方式

自由なアイコンで置きたいなら OJapp が最適

特に BOOTH や note を使っている人は、   サービスのアイコンに統一されてしまい、ブランドが埋もれがちです。

OJapp を使うと、自分のロゴでホーム画面に置いてもらえるため、   認識性・世界観・ブランド性が一気に向上します。
